修改文件名后导致 GitHub 上路径和本地不一致的解决办法

问题描述

在使用 Hexo 部署博客时，修改了文章的文件名后，GitHub 上生成的路径和本地不一致，导致页面无法正确访问。

原因

Hexo 默认使用 .md 文件的文件名作为生成页面的 URL 路径。如果修改了文件名但没有正确同步部署，GitHub Pages 上的路径不会自动更新，导致路径不一致。

解决办法

清除 Hexo 缓存

  1. 删除 .deploy_git 文件夹，强制清除缓存。
  2. 重新执行 Hexo 部署命令：

这样可以确保 Hexo 不会使用旧的缓存记录，重新部署并生成正确的路径。

其实感觉自己找找也可以……不过没必要。

ssh连接方式22端口被封锁解决方案
方案1：切换为 HTTP 连接在当前仓库的 .git/config 文件中，将以下行：1url = git@github.com:username/repo.git修改为：1url = https://github.com/username/repo.git这样我们就可以让 Git 通过 HTTPS 连接 GitHub 仓库。 方案2：切换 SSH 端口在 ~/.ssh 目录下创建或编辑 config 文件，添加以下配置：123456Host github.comUser gitHostname ssh.github.comPreferredAuthentications publickeyIdentityFile ~/.ssh/id_rsaPort 443这样 SSH 就会通过端口 443 连接 GitHub。
