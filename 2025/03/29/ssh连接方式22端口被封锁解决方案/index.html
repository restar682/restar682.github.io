修改文件名后导致github上路径和本地不一致的解决办法
修改文件名后导致 GitHub 上路径和本地不一致的解决办法问题描述在使用 Hexo 部署博客时，修改了文章的文件名后，GitHub 上生成的路径和本地不一致，导致页面无法正确访问。 原因Hexo 默认使用 .md 文件的文件名作为生成页面的 URL 路径。如果修改了文件名但没有正确同步部署，GitHub Pages 上的路径不会自动更新，导致路径不一致。 解决办法清除 Hexo 缓存 删除 .deploy_git 文件夹，强制清除缓存。 重新执行 Hexo 部署命令： 123hexo cleanhexo ghexo d 这样可以确保 Hexo 不会使用旧的缓存记录，重新部署并生成正确的路径。 其实感觉自己找找也可以……不过没必要。
