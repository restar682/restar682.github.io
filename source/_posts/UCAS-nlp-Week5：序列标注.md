# `zip` 与可迭代对象打包

Python 的 `zip()` 函数用于将多个可迭代对象（如列表、元组、字符串等）**并行打包**，返回一个由元组构成的迭代器。每个元组包含各输入可迭代对象在相同位置上的元素，适用于需要**同步遍历多个序列**的场景。

- `zip(iter1, iter2, ..., iterN)`：接收任意数量的可迭代对象，返回一个 `zip` 对象（迭代器）。当最短的输入可迭代对象耗尽时，`zip` 停止生成新元组。
- `list(zip(...))`：常通过 `list()` 将 `zip` 对象转为列表以便查看或复用，例如 `list(zip([1,2], ['a','b']))` 得到 `[(1, 'a'), (2, 'b')]`。
- `zip(*iterable)`：在调用时使用 `*` 对一个包含多个序列的容器进行**解包**，可以实现“转置”效果。  
  例如，若 `pairs = [(1, 'a'), (2, 'b')]`，则 `zip(*pairs)` 等价于 `zip((1, 'a'), (2, 'b'))`，可解包还原为 `(1, 2)` 和 `('a', 'b')`。

> 注意：`zip` 返回的是迭代器，只能遍历一次；如需多次使用，应转换为列表或元组。

# `namedtuple` 与具名元组
Python 的 `collections.namedtuple` 可以创建**带有字段名的元组子类**。它在保留普通元组不可变性和高效性的基础上，还允许通过**属性名**访问元素，从而显著提升代码的可读性。

- `namedtuple(typename, field_names)`：
  - `typename`：新元组子类的名称（如 `'Point'`）；
  - `field_names`：字段名序列，可为字符串（如 `'x y'`）、列表（如 `['x', 'y']`）或元组。
    该函数返回一个 **可实例化的类**，其实例既是元组，又支持通过属性名访问字段。

`namedtuple` 的实例与普通元组一样是**不可变的**，内存开销接近原生元组，通常远低于常规类实例。常用方法包括：

- `_make(iterable)`：**类方法**，通过可迭代对象创建实例，例如 `Point._make([3, 4])`；
- `_asdict()`：**实例方法**，将实例转换为字典，例如 `p._asdict()` 返回 `{'x': 1, 'y': 2}`；
- `_replace(**kwargs)`：**实例方法**，返回一个新实例（原实例不变），用于“更新”指定字段，例如 `p._replace(x=10)`。
